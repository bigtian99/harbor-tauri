# OPS 版 vs 完整版

## 构建方式

| 产物 | 命令 / 方式 |
|------|-------------|
| 完整版 | `pnpm tauri:build`（及 arm64/x64/universal 变体） |
| OPS 版 | `pnpm tauri:build:ops` / `pnpm tauri:build:ops:win64`（`OPS_MODE=true`） |

后端：`is_ops_mode()` 在编译期读 `OPS_MODE`；为 `true` 时前端隐藏非运营菜单。

## 菜单 / 能力对照（以当前前端侧边栏为准）

| 能力 | 完整版 | OPS 版 |
|------|--------|--------|
| 上传推送 | ✅ | 通常隐藏 |
| 镜像推送 | ✅ | 通常隐藏 |
| 分支打包 | ✅ | 通常隐藏 |
| 分支合并 | ✅ | 视实现 |
| 历史记录 | ✅ | 视实现 |
| 生成落地页 | ✅ | ✅ 核心 |
| 结算单 | ✅ | ✅ 核心 |
| 打包加速 | ✅ | ✅ 核心 |
| 设置（Harbor 等） | ✅ | 部分保留（按前端 `opsMode` 逻辑） |
| 系统日志 | ✅ | ✅ |

> 若前端 `opsMode` 裁剪列表变更，请同步更新本表。

## 注意

- 用户反馈「某菜单不见了」时先确认是否装了 OPS 包。
- 开发调试完整功能请用非 OPS 构建。
